# Interactions

Source: [backend/src/movie_recommender/api/v1/interactions.py](../../backend/src/movie_recommender/api/v1/interactions.py)

Register a user's swipe on a movie. One endpoint, three side effects.

| Method | Path | Auth | Body | Returns |
|---|---|---|---|---|
| POST | `/api/v1/interactions/{movie_id}/swipe` | required | `SwipeRequest` | `RegisteredFeedback` |

---

## Request — `SwipeRequest`

```json
{
  "action_type": "like",
  "is_supercharged": false
}
```

`action_type` ∈ `"like"`, `"dislike"`, `"skip"`. `is_supercharged` is optional, default `false`.

> **Constraint:** `action_type="skip"` + `is_supercharged=true` → **400** `"Cannot have 'SKIP' interaction supercharged"`.

> **Semantics:** down-swipe in the UI maps to `skip`, not `dislike`. `skip` is *neutral* ("not interested right now"); the recommender treats it as a softer negative signal than `dislike`.

---

## What the endpoint does

1. Validate the movie exists in Postgres.
2. **Fire-and-forget** persist the swipe via `asyncio.create_task(persist_swipe(...))` (the swipe worker writes to `swipes`).
3. Synchronously update the recommender model (`recommender.set_user_feedback(...)`) — this is the online-learning path that mutates the user's vector.
4. **Fire-and-forget** update the Neo4j knowledge-graph beacon map (cast/director/genre weights).

Because (2) and (4) are fire-and-forget, a **200** does not guarantee that the row has landed in `swipes` by the time the response returns. The response confirms enqueue, not durability.

---

## Response — `RegisteredFeedback`

```json
{
  "interaction_id": 0,
  "movie_id": 42,
  "action_type": "like",
  "is_supercharged": false,
  "registered": true
}
```

`interaction_id` is currently always `0` (the row id is generated by the async worker, not returned synchronously).

---

## Errors

| Status | Body | Cause |
|---|---|---|
| 400 | `{"detail":"Cannot have 'SKIP' interaction supercharged"}` | Invalid action+supercharge combo |
| 401 | `{"detail":"Authentication failed"}` | Missing/invalid token |
| 404 | `{"detail":"User not found"}` | Token valid, no `users` row matches |
| 404 | `{"detail":"Movie not found"}` | `{movie_id}` does not exist in `movies` |
